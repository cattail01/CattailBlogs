## 后记

该项目或许标志着我将彻底更改自己的学习方式。

以前我的学习模式非常符合应试教育。从课文背诵，到英文单词一个不拉的默写，仿佛用牛刀杀鸡，用大炮打蚊子。到了高二开始，可能是因为自己知道的原因，我开始强迫自己整理所有科目的学习思维导图和笔记，贪婪地希望自己的笔记能做到尽善尽美。但是换来的却是痛苦，完美主义的痛苦，和理论脱离实践的痛苦。

然后理所应当地，我没能进入真正的更高等级的学府进行深造。大学四年过的浑浑噩噩的，不过有两件事一直没有放弃：学习计算机编程语言，和找更加普适性的学习方法。从机器学习原理，到学习成功论，我应该是翻了不下几百篇交别人如何学习的博客和视频。走过不少弯路，（还被带偏到mbti的坑里）。

转变的根源是我大学里一个社团的哥们，是我认识的大佬之一。我研究了他心理整整3年之久，发现他整个思维都是逆向的：从问题出发，利用自己的知识和其他一切资源手段，向答案推进。而我的思维方式却正好相反，从自己已经学过的知识点出发，结合问题，向问题逐步推进。我这一套虽然对于应试教育是有效的，但是对于计算机学习却收效甚微了；而且由于高中过于重视知识点的学习，忽略了练习强化，以致连应试都失败了。

最后总之，对于计算机编程、建模、游戏编程和单片机编程等以实践为主的项目进行学习时，应当采用项目驱动知识点反馈，辅以知识点框架的方法，而不是追求学完完整知识点后再进行实际项目学习。而对于考研考公务员考各种证件等应试，以及一些知识性为主的学科学习，比如心理学、营养学、金融学等，应当在搭建学习知识点框架（过知识点）的同时，将刷题、实践与之结合，甚至做到3分学知识和手法，4分实践与练习，3分解决练习出现的问题和纠正问题。这样或许是最佳学习方法。

但是不论用那种方法，都应当低下骄傲的头颅，多虚心参考别人的成果，多听别人的讲解。“君子生非异也，善假于物也。“同时我希望能够利用好我能够得到的一切有利资源。或许能够使我达到大佬的一半水平，这样就足够生存，并达到我想要的目标了。

希望你我的学习之旅一帆风顺。
